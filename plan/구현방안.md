## 2. 구성요소

### A. 데이터 전처리

| 구성요소 | 설명 |
|---------|------|
| 품질 필터링 | mean intensity 기반 dark/bright/low contrast 이미지 탐지 및 분류 |
| CLAHE | 대비 제한 적응형 히스토그램 균등화로 폐 병변 시인성 개선 |
| Resize + 정규화 | 224x224 + ImageNet 평균/표준편차 정규화 |
| Multi-hot Encoding | 14개 질환에 대한 이진 벡터 생성 |
| Patient-wise Split | Patient ID 기준 Train/Test 85:15 분리, 5-Fold GroupKFold |

### B. 모델 학습

| 구성요소 | 설명 |
|---------|------|
| DenseNet-121 | ImageNet Pretrained, Focal Loss gamma 0/1/2 실험 |
| EfficientNet-B0/B4 | ImageNet Pretrained, 최적 gamma로 학습 |
| Focal Loss | 직접 구현 (외부 라이브러리 미사용), pos_weight로 클래스 불균형 보정 |
| 학습 전략 | AdamW + Cosine Annealing + Early Stopping (patience=5, val_auroc) |
| 교차 검증 | 5-Fold GroupKFold, 환자 겹침 없음 검증 |
| 앙상블 | 모든 fold × 모델 조합 brute force 탐색, 최적 조합 선정 |

### C. 성능 평가 및 분석

| 구성요소 | 설명 |
|---------|------|
| AUROC / AUPRC | 14개 질환별 + Mean, Fold별 Mean ± Std |
| TTA (Test-Time Augmentation) | 추론 시 H-Flip 적용, 전후 AUROC 비교 |
| Youden's J Threshold | 질환별 최적 Cut-off 산출 |
| Operating Point 분석 | Sens@Spec90, Spec@Sens90 계산 |
| Calibration | ECE 계산 + Temperature Scaling (ECE > 0.10 시) |
| Subgroup Analysis | 성별/연령대/View Position별 AUROC 비교 |
| External Validation | CheXpert 또는 PadChest에서 Domain Shift 분석 |
| Grad-CAM | 마지막 Conv Layer 기반 히트맵 생성 |
| 에러 분석 | FP/FN 케이스 Grad-CAM 분석, Shortcut Learning 검출 |

### D. 모델 서빙 API

| 구성요소 | 설명 |
|---------|------|
| FastAPI 서버 | /health, /predict 엔드포인트 |
| 추론 파이프라인 | 이미지 → 전처리 → 2-Model Ensemble → 확률 + Grad-CAM |
| Pydantic 스키마 | 입출력 데이터 검증 |
| 에러 핸들링 | 잘못된 이미지 포맷 처리 |
| 성능 요구사항 | 1장당 500ms 이내 (CPU, 2-Model best fold) |

### E. 판독 보조 대시보드

| 구성요소 | 설명 |
|---------|------|
| Streamlit 앱 | 이미지 업로드 + 결과 시각화 |
| 확률 시각화 | 14개 질환 수평 막대그래프, 위험도 색상 (빨/노/초) |
| Grad-CAM 시각화 | 원본 + 히트맵 오버레이 |
| API 통신 | 모델 직접 로드 X, FastAPI 서버로 HTTP 요청 |

### F. 개발 환경 및 테스트

| 구성요소 | 설명 |
|---------|------|
| Dockerfile | API 서버 실행 환경 (CPU 기반) |
| pytest | 전처리, Multi-hot Encoding, API 응답 스키마 테스트 (3개 이상) |
| README.md | 실행 방법, 성능 표, 분석 결과 |
